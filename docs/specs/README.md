# Philosophies Of A Cyborg Specs

This directory holds the product and feature specs for the first public version
of **Philosophies Of A Cyborg**: a personal publication for Ositu Kengere's writing,
thinking, research, and experiments.

The standard is not "a blog template with posts." The standard is a publication
that feels authored, fast, durable, memorable, and unusually alive.

## Product Principles

1. **The writing is the product.** Visual design must sharpen attention, not
   perform circus tricks over the prose.
2. **The site should feel like Ositu Kengere.** Direct, intelligent, funny,
   rough where it needs to be, and allergic to beige internet sludge.
3. **Fast by default.** Reading pages must load quickly, stay stable, and work
   without JavaScript where possible.
4. **Built for a long archive.** The system should still make sense after 500
   posts, notes, essays, links, and experiments.
5. **Editorial control beats automation.** AI can assist, but final structure,
   tone, and publication state are deliberate.
6. **Every feature earns its place.** If it does not improve reading, writing,
   discovery, or identity, it is decoration and probably guilty.

## First Build Outcome

The first build should deliver:

- A polished public home page.
- Essay, note, and link post types.
- Topic and series pages.
- A premium long-form reading template.
- Search and archive navigation.
- Draft-to-publish content workflow.
- RSS, metadata, sitemap, and social preview support.
- A technical base ready for future interactive/AI features.

## Recommended Build Order

1. Technical foundation and content schema.
2. Information architecture and route generation.
3. Reading experience.
4. Home, archive, topic, and series pages.
5. Search and related reading.
6. Design polish, accessibility, performance, and metadata.
