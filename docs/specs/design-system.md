# Design System

## Summary

The visual system should feel like a serious personal publication with a cyborg
edge: restrained, precise, slightly dangerous, and built around prose. It should
avoid generic dark-mode SaaS mush, purple gradient sludge, and decorative blobs
that scream "I discovered CSS last Tuesday."

## Design Direction

Keywords:

- Editorial.
- Mechanical.
- Human.
- Sharp.
- Calm.
- Archival.
- Slightly ominous, never gimmicky.

## Core Surfaces

- Home page.
- Article page.
- Listing page.
- Topic page.
- Series page.
- Archive page.
- About page.

## Components

- Site header.
- Mobile nav.
- Post card/list item.
- Topic chip.
- Series progress block.
- Article header.
- Table of contents.
- Footnote/reference block.
- Code block.
- Callout.
- Related reading list.
- Search/filter controls.

## Color And Theme

Requirements:

- Strong light mode.
- Strong dark mode if included in V1.
- Avoid one-note palettes.
- Use accent color sparingly for navigation, links, and emphasis.
- Body copy contrast must meet WCAG AA.

## Layout Rules

- No nested cards.
- No giant landing-page hero unless the content truly benefits from it.
- No decorative orbs/blobs.
- Article pages should use unframed prose layouts, not cards inside cards.
- Cards are only for repeated preview items or contained tools.

## Acceptance Criteria

- The home page has a clear first-viewport identity signal.
- Article text is the visual priority.
- Mobile layout has no clipped navigation, overlapping headings, or awkward
  button text.
- The design works with real long titles and uneven content lengths.
